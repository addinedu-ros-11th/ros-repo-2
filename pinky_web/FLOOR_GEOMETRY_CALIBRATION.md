# Pinky Floor Geometry Calibration (No LiDAR)

## 목적
- 카메라 기반 바닥 주행 1단계에서 반복 계산 없이 동일 기준을 사용한다.
- 픽셀 좌표를 바닥 평면의 근사 거리/오프셋으로 변환할 때 기준점으로 사용한다.

## 고정 전제
- 스트림 해상도: `320x240`
- 카메라 높이: 바닥 기준 `60 mm`
- 카메라 자세(기울기/장착) 고정
- 카메라 flip 설정 고정 상태에서 측정

## 확정 캘리브레이션 값
1. 하단 중앙점(bottom center)
- 화면 맨 아래 중앙의 바닥 직선거리(평면): `R = 90 mm`
- 중심선 점이므로 `(X, Y) = (90, 0)`으로 둔다.

2. 하단 좌우 끝점(bottom left/right)
- 화면 맨 아래 좌/우 끝점의 바닥 직선거리(평면): `R = 107 mm`
- 하단 행은 동일 전방거리(`X_bottom`)를 공유한다고 가정:
  - `X_bottom = 90 mm`
- 따라서 좌/우 오프셋은:
  - `Y_edge = sqrt(107^2 - 90^2) = 57.8 mm`
  - 좌/우 끝점: `(X, Y) = (90, -57.8)`, `(90, +57.8)`
- 하단 실폭:
  - `W_bottom = 2 * 57.8 = 115.6 mm`

3. 화면 정중앙(center pixel)
- 화면 정중앙 `(u, v) = (cx, cy)`의 바닥 직선거리(평면): `R = 300 mm`
- 중심선 점이므로 `(X, Y) = (300, 0)`으로 둔다.
- 좌우 오프셋: `Y = 0 mm`

4. 특이선(수평선 근처)
- 카메라 높이와 같은 높이(약 60mm) 지점은 화면에서 거의 일정한 높이에 나타남
- 관측값: 바닥 기준 `75%` 지점(from bottom)
  - 240px 높이 기준 대략 `v_h ~ 60` (top 기준, `180` from bottom)
- 거리 계산 안정화를 위해 ROI 시작선 권장:
  - `v_start = v_h + 10 px` (기하 기준 최소 시작선)

## 좌표/표기
- 픽셀: `(u, v)` (`u`: 가로, `v`: 세로)
- 바닥 평면 좌표:
  - `X`: 전방(+)
  - `Y`: 좌우(+우, -좌)

## 하단 라인 전용 근사식 (v = bottom row)
- 화면폭 `W=320`, 중심 `c=(W-1)/2 ~= 159.5`
- 정규화: `t = (u-c)/(W/2)` (`-1 ~ +1`)

1. 좌우 오프셋
- `Y(u) = 57.8 * t` (mm)

2. 전방거리
- 하단 행은 전방거리 상수로 둔다:
  - `X(u) = 90` (mm)
- 이 식은 bottom row 근사식이며, 전체 행(v) 확장은 별도 LUT/모델로 진행

## 실무 규칙
- 바닥 연산(거리/조향)에서는 상단 `1/3` 영역을 제외한다.
  - 240px 높이 기준: `v >= 80`만 사용
- 바닥/조향 계산은 위 ROI와 `v >= v_start`를 함께 만족하는 영역만 사용한다.
- 캘리브레이션 변경 시(카메라 높이/기울기/해상도 변경) 이 문서를 업데이트한다.

## 빠른 검산 체크
- bottom center: `(X, Y) = (90, 0)`
- bottom left/right: `(90, -57.8)`, `(90, +57.8)`
- `R_left/right = sqrt(90^2 + 57.8^2) ~= 107 mm`
- center pixel: `(300, 0)` (`R=300 mm`)
- 특이선: 화면 높이 기준 bottom에서 `75%` 지점에서 거리 발산(무한대 근사)
